# 🔹 FASE 4 – Businessmap Mastery (Essencial para sua realidade)

Na Zello usamos o businessmap para organização de demandas, que é essencial para que o time se organize e os coordenadores cadastrem demandas que cada um do squad está participando.

---

## 🔸 Estrutura que você precisa dominar completamente

### 1ª Fileira – Clientes / Projetos

Relaciona cliente ↔ OS ↔ faturamento (página 2)

Você precisa:

- Garantir que toda OS esteja vinculada corretamente
    
- Evitar perda de rastreabilidade
    

---

### 2ª Fileira – OS de Concepção e Execução (página 3)

Você deve dominar:

- Separação clara:  
    OS Concepção  
    OS Execução
    
- Nunca misturar
    

---

### 3ª Fileira – Atividades Técnicas (página 4)

Aqui você atua como orquestrador.

Criar atividades separadas por:

[Frontend] TEMA | OS 2026-XXXX  
[Backend] TEMA | OS 2026-XXXX  
[BD] TEMA | OS 2026-XXXX  
[QA] TEMA | OS 2026-XXXX

Conforme padrão de nomenclatura (página 5)

---

## 🔸 Hierarquia (página 7)

Você deve sempre garantir:

Cliente (avô)  
↳ OS (pai)  
↳ Atividades (filhos)

Se quebrar essa hierarquia, você perde controle estratégico.

---

## 🔸 Subtasks (página 8)

Você deve usar subtasks para:

- Rastrear micro atividades
    
- Distribuir responsabilidade
    
- Facilitar QA
    

---

## 🔸 Tipos de Cards (página 10)

Muito importante:

- Sub-task = atividade
    
- Defect = erro de homologação
    
- Bug = erro em produção
    
- História de Usuário = escopo funcional
    

Isso impacta análise de qualidade do sistema.

---

## 🔸 Owner / Co-owner (página 11)

Você deve garantir:

- Responsável claro
    
- Corresponsável quando houver apoio
    

Isso evita ruído político e técnico.

---

## 🔸 Deadline / Size / Tags (página 12)

Você precisa usar isso estrategicamente:

- Deadline = compromisso
    
- Size = previsão (horas)
    
- Logged Time = custo real
    
- Tags = visão analítica
