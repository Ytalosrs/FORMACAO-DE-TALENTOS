
Na Mulesoft:

Feature = Características
Fast-paced = Acelerado / Ágil
Timely = Oportuno / No tempo certo
Up-to-date = Recente / Informações mais Recentes
Siloed = Isolado 
Shrinking = encolhendo / diminuir
batch = Lote ou barras - Armazenar em lote
scratch - do zero - Começar do zero
zero-downtime: ausência total de interrupções
on-premises = no loca / instalação própia
regardless = Apesar de que
syncing = sincronizar
UAT = User Acceptance Testing Teste de Aceitação do Usuário

primary target user:  é um termo para indicar **para quem** aquela ferramenta foi feita, qual é o papel/perfil principal de usuário que deve usá‑la (admin, business user, dev de integração, etc.).

Sample data =  conjunto pequeno de dados de exemplo, usado só para demonstrar, testar ou configurar uma integração/app, sem depender dos dados reais de produção.

rule-based =  mix de escalas horizontal/vertical
