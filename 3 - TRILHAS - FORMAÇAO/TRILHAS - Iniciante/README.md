- Documentação de Desafio
- Nome do desafio
Aprendizado e Utilização do Git e GitHub

### Objetivo

Aprender a utilizar o Git e o GitHub para realizar o versionamento de código, gerenciar repositórios e aplicar boas práticas de desenvolvimento colaborativo.

### Descrição

O desafio consistiu em aprender os conceitos fundamentais de Git e GitHub, desde a criação e organização de repositórios até o envio de projetos para a plataforma. Durante o processo, também foram realizadas alterações na estrutura dos repositórios, como renomeação de arquivos e pastas, organização do conteúdo e utilização dos principais comandos de versionamento.

### Principais desafios encontrados
- Compreender o funcionamento do versionamento de código.
- Aprender a utilizar os principais comandos do Git.
- Organizar corretamente os arquivos dentro dos repositórios.
- Renomear arquivos e pastas mantendo a estrutura do projeto.
- Excluir diretórios desnecessários sem comprometer o repositório.
- Sincronizar corretamente as alterações entre o computador e o GitHub.
  
### solução adotada

Para superar os desafios, foram realizados estudos por meio de cursos e atividades práticas. Cada conceito aprendido foi aplicado diretamente em repositórios de teste, permitindo compreender o fluxo de trabalho do Git e do GitHub e desenvolver familiaridade com os comandos mais utilizados.

### resultados obtidos

Ao final do desafio, foi possível utilizar o Git e o GitHub para criar e organizar repositórios, controlar versões dos projetos, realizar commits, enviar alterações para o repositório remoto e gerenciar a estrutura dos arquivos. Também foi adquirido conhecimento sobre a organização e manutenção de projetos utilizando ferramentas de versionamento.
